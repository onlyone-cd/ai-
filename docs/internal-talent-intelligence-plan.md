# 内部人才智能分析模块改进方案

## 1. 背景与目标

现有系统主要面向外部招聘：简历上传、人才库解析、岗位 JD 管理、人岗匹配、面试、Offer、BOSS 闭环。

本次新增方向是把公司内部在职员工纳入统一人才分析体系：

- 上传公司内部员工简历或员工能力档案。
- 建立公司组织架构，按公司、部门、团队查看内部员工。
- 导入公司所有岗位 JD。
- 录入或导入员工当前薪资、职级、部门、岗位、在职状态。
- 使用 AI 和规则模型分析：
  - 员工能力与当前岗位 JD 是否匹配。
  - 员工薪资与能力、岗位、市场/内部区间是否合理。
  - 员工离职后，是否需要从外部人才库推荐相似替补人选。
  - 员工调岗时，推荐更适合他的内部岗位。

核心目标不是简单打分，而是帮助 HR / 管理者回答四类问题：

1. 这个员工是否适合当前岗位？
2. 这个员工的薪资是否合理？
3. 如果这个员工离职，谁可以替代？
4. 如果这个员工想调岗，最适合哪个岗位？

## 2. 业务对象定义

### 2.1 内部员工

内部员工与外部候选人不同，需要新增独立身份字段，但可复用简历解析、技能标签、岗位匹配能力。

建议字段：

- 员工姓名
- 手机号/邮箱
- 员工编号
- 部门
- 当前岗位
- 当前职级
- 当前薪资
- 薪资结构：月薪、年终、年包、补贴、股权等，第一期可只做月薪/年包
- 入职日期
- 在职状态：在职、离职、待离职、调岗中
- 直属上级
- 员工简历/能力档案原文
- 解析后的教育、工作经历、项目经历、技能标签、能力雷达
- 数据来源：手动上传、Excel 导入、HR 系统同步

### 2.2 内部岗位 JD

内部岗位可复用当前岗位模块，但需要补充内部管理字段。

建议字段：

- 岗位名称
- 岗位编号
- 部门
- 城市
- 职级范围
- 薪资范围
- JD 原文
- 技能权重
- 必备能力
- 加分能力
- 经验年限
- 学历要求
- 岗位状态：开放、冻结、内部调岗开放、仅外招、关闭

### 2.3 内部薪资基准

为了判断薪资是否合理，不能只看员工个人薪资，还需要岗位/职级的参考区间。

第一期可以先用岗位 JD 上的薪资范围作为基准。

后续可扩展：

- 公司内部职级薪资带宽
- 部门薪资带宽
- 城市薪资差异
- 市场薪资数据
- 绩效等级
- 任职年限

### 2.4 公司组织架构

组织架构是内部人才模块的核心入口，用于按照真实公司层级查看部门、团队和员工。

建议字段：

- 组织节点名称
- 组织节点类型：公司、事业部、部门、小组
- 上级组织节点
- 部门负责人
- HRBP
- 所属城市
- 编制人数
- 当前在职人数
- 空缺人数
- 组织状态：正常、合并、撤销、冻结

组织架构需要支持树形结构，例如：

- 总公司
- 技术中心
- 后端开发部
- Java 组
- AI 应用组
- 产品中心
- 招聘部
- 财务部

员工需要绑定到具体组织节点，而不是只记录一个文本部门名。这样后续才能做部门维度的统计、权限控制和人才盘点。

### 3.2 当前岗位匹配分析

输入：

- 员工简历解析结果
- 员工当前岗位
- 当前岗位 JD
- 岗位技能权重

输出：

- 当前岗位匹配分，满分 100。
- 命中技能。
- 缺失技能。
- 风险项。
- AI 解释。
- 是否建议：
  - 继续当前岗位
  - 需要培训补齐能力
  - 建议调岗
  - 不适配

初步规则：

- 70%：JD 技能权重与员工技能标签匹配。
- 15%：经验年限、项目经历、行业背景。
- 10%：学历/证书/语言等硬性条件。
- 5%：岗位稳定性和成长潜力，由 AI 结合简历文本判断。

分数解释：

- 85-100：高度匹配，可作为核心人才或继任候选。
- 70-84：基本匹配，建议继续任用，可补齐短板。
- 50-69：部分匹配，需要培训或重新评估岗位适配。
- 50 以下：不建议当前岗位继续匹配，可优先考虑调岗。

### 3.3 薪资合理性分析

输入：

- 员工当前薪资
- 当前岗位薪资范围
- 职级
- 城市
- 匹配分
- 员工能力标签和经验

输出：

- 薪资合理性评分，满分 100。
- 薪资状态：
  - 明显偏低
  - 略低
  - 合理
  - 略高
  - 明显偏高
- AI 分析说明。
- 调薪建议：
  - 暂不调整
  - 建议小幅调整
  - 建议重点保留
  - 建议复核薪资与绩效

第一期规则建议：

- 如果员工薪资低于岗位薪资下限，同时岗位匹配分高于 80，标记为“偏低，高保留风险”。
- 如果员工薪资高于岗位薪资上限，同时岗位匹配分低于 60，标记为“偏高，需复核产出”。
- 如果薪资在岗位区间内，且匹配分高于 70，标记为“合理”。
- 如果岗位没有薪资范围，则只给“数据不足”，不强行判断。

注意：

- 薪资是高敏感数据，需要权限控制。
- 普通 HR 可看分析结果，薪资明细建议只允许管理员/经理角色查看。
- 所有薪资查看、导出、AI 分析都需要进入审计日志。

### 3.4 离职替补推荐

场景：

当员工状态变为“离职”或“待离职”时，系统自动分析该员工能力画像，并从外部人才库中推荐相似候选人。

输入：

- 离职员工能力标签
- 离职员工当前岗位 JD
- 离职员工项目经历
- 外部人才库候选人

输出：

- 推荐替补候选人列表。
- 替补匹配分。
- 与离职员工相似点。
- 与岗位 JD 的匹配点。
- 风险项。
- 推荐优先级。

推荐逻辑：

- 先匹配岗位 JD，保证候选人能胜任岗位。
- 再匹配离职员工的技能画像，找类似能力结构。
- 最后结合经验、城市、薪资期望等因素排序。

第一期可复用现有岗位匹配能力：

- 离职员工当前岗位 JD -> 外部人才库匹配。
- 离职员工技能画像 -> 相似度补充排序。

### 3.5 内部调岗推荐

场景：

HR 想知道某个员工除了当前岗位之外，更适合公司哪个岗位。

输入：

- 员工简历/能力标签
- 公司全部开放内部调岗的岗位 JD
- 岗位薪资范围、职级范围

输出：

- 推荐岗位列表。
- 每个岗位匹配分。
- 推荐理由。
- 能力差距。
- 是否建议培训后转岗。

排序规则：

- 只展示匹配分 >= 50 的岗位。
- 85 分以上标记为“强推荐”。
- 70-84 标记为“可转岗”。
- 50-69 标记为“培训后可考虑”。

调岗推荐需要排除：

- 当前岗位。
- 已关闭岗位。
- 不开放内部调岗的岗位。
- 职级跨度过大的岗位，除非管理员允许。

## 4. AI Agent 能力扩展

AI 助手需要支持自然语言查询和分析，例如：

- “分析张三当前岗位匹配度和薪资是否合理。”
- “如果李四离职，推荐 5 个可以替代他的候选人。”
- “王五适合调到哪些岗位？”
- “列出薪资偏低但岗位匹配分高于 85 的员工。”
- “哪些员工岗位不匹配但薪资偏高？”
- “研发部哪些员工有离职替补风险？”

Agent 不应只返回明细列表，要返回结论：

- 关键结论
- 推荐动作
- 证据
- 可点击的员工/岗位/候选人

## 8. 权限与安全

薪资和在职员工信息属于高敏感数据，必须比外部候选人更严格。

建议权限：

- admin：全部可见、可导入、可导出、可分析。
- manager：可看本部门及下级部门员工，薪资是否显示由配置决定。
- recruiter：可看基础信息和推荐结果，不默认显示薪资明细。
- interviewer：不可访问内部员工薪资模块。

审计要求：

- 查看部门员工列表
- 查看员工详细简历
- 查看员工薪资
- 导入薪资
- 导入组织架构
- 修改组织架构
- 导出员工数据
- AI 分析薪资合理性
- 查看离职替补推荐

都必须写入审计日志。

## 10. MVP 开发顺序

### 第一阶段：内部员工基础库

- 新增组织架构表和组织架构树。
- 新增内部员工模块。
- 支持按部门查看员工。
- 支持上传员工简历。
- 支持员工基础信息维护。
- 支持薪资字段录入。
- 支持员工详情页。

### 第二阶段：当前岗位匹配与薪资分析

- 复用现有人岗匹配规则。
- 员工当前岗位与 JD 匹配。
- 薪资合理性分析。
- 生成员工分析报告。

### 第三阶段：离职替补与调岗推荐

- 员工离职后推荐外部候选人。
- 根据员工能力推荐内部岗位。
- 支持批量分析。

### 第四阶段：AI Agent 与 BI

- AI 助手支持内部员工查询。
- BI 增加内部人才风险看板。
- 支持薪资风险、离职替补风险、岗位不匹配风险统计。

## 11. 需要确认的问题

请确认以下口径后再开始开发：

1. 模块名称用“内部人才”还是“在职员工”？
2. 内部员工是否和外部候选人共用人才库，还是单独建“内部员工库”？
3. 组织架构第一期是否需要支持 Excel 导入？
4. 组织架构是否需要支持公司/事业部/部门/小组四级，还是只做部门/小组？
5. 部门负责人是否也是内部员工？
6. 薪资第一期只做月薪/年包，还是需要奖金、补贴、股权？
7. 薪资是否允许 recruiter 角色查看？
8. 调岗推荐是否只推荐“内部调岗开放”的岗位？
9. 离职替补推荐是只从外部人才库推荐，还是也允许推荐内部员工替岗？
10. 员工数据是否需要 Excel 模板导入？
11. 是否需要导出完整分析报告？
